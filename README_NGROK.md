# Настройка ngrok для работы с мини-приложениями

## Описание

ngrok используется для создания публичного туннеля к локальному API серверу, чтобы мини-приложения на Vercel могли к нему обращаться.

## Быстрый старт

1. **Запустите API сервер:**
   ```bash
   python -m src.api.main
   ```

2. **В другом терминале запустите ngrok:**
   ```bash
   make ngrok_start
   ```
   Или напрямую:
   ```bash
   ./scripts/start_ngrok.sh
   ```

3. **Обновите мини-приложения с ngrok URL:**
   ```bash
   make ngrok_url
   ```
   Или напрямую:
   ```bash
   python3 scripts/get_ngrok_url.py
   ```

4. **Теперь мини-приложения будут использовать ngrok URL для API запросов!**

## Команды

- `make ngrok_start` - Запустить ngrok туннель
- `make ngrok_url` - Получить текущий ngrok URL и обновить мини-приложения
- `make ngrok_stop` - Остановить ngrok туннель

## Как это работает

1. Скрипт `start_ngrok.sh` запускает ngrok и сохраняет URL в файл `.ngrok_url`
2. Скрипт `get_ngrok_url.py` читает URL из ngrok API или файла и обновляет meta-теги в мини-приложениях
3. Мини-приложения читают URL из meta-тега `<meta name="api-base-url" content="...">`

## Важно

- ngrok URL меняется при каждом перезапуске (если не используете платный план)
- После перезапуска ngrok нужно снова выполнить `make ngrok_url`
- Для продакшена лучше развернуть API на публичном домене

## Альтернативы

- **ngrok (бесплатный)** - URL меняется при перезапуске
- **ngrok (платный)** - Статический домен
- **Cloudflare Tunnel** - Бесплатный статический домен
- **Развертывание API на сервере** - Лучший вариант для продакшена
